Readme - Package Level Documentation.

Package: Cataphract.API
Author: DAK404 (https://github.com/DAK404)

-[ INTRODUCTION ]-

The Cataphract.API package provides a comprehensive set of APIs that facilitate various functionalities within the Cataphract OS Shell. These APIs are designed to interact with different components of the system, enabling seamless integration and operation.

-[ DETAILS ]-

The Cataphract.API package is divided into several sub-packages, each serving a specific purpose:

* Astaroth - The Astaroth sub-package includes classes and methods for advanced system operations and management. It provides tools for system diagnostics, performance monitoring, and other critical tasks.

* Dragon - The Dragon sub-package handles user authentication and security. It includes classes for login management, privilege checking, and other security-related functionalities.

* Minotaur - The Minotaur sub-package is responsible for policy management. It provides classes and methods to enforce system policies, ensuring that the system operates within defined parameters.

* Wraith - The Wraith sub-package deals with file management and operations. It includes utilities for file reading, writing, downloading, and unzipping, among other file-related tasks.

* Wyvern - The Wyvern sub-package provides utilities for updating the Cataphract system. It includes classes for downloading and installing updates, ensuring that the system remains up-to-date with the latest features and security patches.

The classes currently implemented in this package are:

    * Anvil.java
    * Build.java
    * ExceptionHandler.java
    * IOStreams.java

The following details the functionality of each class:

* Anvil.java: Provides basic features and functionalities for the Cataphract system.

* Build.java: Handles the build process and related operations.

* ExceptionHandler.java: Manages exceptions and error handling within the system.

* IOStreams.java: Provides input and output stream utilities for the system.
